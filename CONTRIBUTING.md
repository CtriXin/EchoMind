# Contributing

Thanks for contributing to EchoMind. Keep changes focused and easy to review.

## Local Setup

Install dependencies from the repository root:

```bash
npm install
```

## Tests

Run the test suite before opening a pull request:

```bash
npm test
```

For a quick CLI smoke check, run:

```bash
npm run smoke
```

## Pull Requests

- Create a branch with a short, descriptive name, such as `docs/add-contributing` or `fix/issue-123`.
- Use concise commit messages that describe the change; prefer conventional prefixes like `docs:`, `fix:`, or `feat:` when they fit.
- In the PR description, summarize what changed, why it changed, and how you validated it. Link the related issue when applicable.
