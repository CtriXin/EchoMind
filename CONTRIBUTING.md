# Contributing

Thanks for contributing to EchoMind. Keep changes focused and easy to review.

## Local Setup

EchoMind requires Node.js 18.17 or newer.

Install dependencies from the repository root:

```bash
npm install
```

## Tests

EchoMind does not have a formal test suite yet. For a quick CLI smoke check that runs `echomind help`, use:

```bash
npm run smoke
```

## Issues

Before opening a pull request, check for an existing issue or open one with the bug, feature request, or documentation gap you want to address.

## Pull Requests

- Create a branch with a short, descriptive name, such as `docs/add-contributing` or `fix/issue-123`.
- Use concise commit messages that describe the change; prefer conventional prefixes like `docs:`, `fix:`, or `feat:` when they fit.
- In the PR description, summarize what changed, why it changed, and how you validated it. Link the related issue when applicable.

## Code of Conduct and License

Be respectful and constructive in project discussions. Contributions are accepted under the repository's MIT license.
